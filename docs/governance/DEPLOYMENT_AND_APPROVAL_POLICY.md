# Política de Deploy e Aprovação

## Objetivo

Definir limites para uso de Git, Vercel, Railway e AI Bridge Local.

## Ações que exigem aprovação

- commit
- push
- pull request merge
- deploy Vercel
- deploy Railway
- alteração de variável de ambiente
- alteração de banco de dados
- alteração de campanha paga
- envio de mensagem externa
- acesso a secrets
- execução destrutiva

## Ações permitidas sem aprovação

- ler arquivos
- gerar documentação
- criar relatórios
- listar agentes
- criar tarefas locais
- rodar validações read-only
- sugerir patches

## Modo inicial do escritório

O modo inicial deve ser assistido.

Agentes podem planejar e sugerir.

Execução real deve passar por gates.

Produção deve exigir confirmação humana.
