# Runbook Operacional

## Rotina padrão

1. Receber demanda.
2. Classificar risco.
3. Delegar para agente adequado.
4. Executar em modo seguro.
5. Validar resultado.
6. Registrar logs.
7. Enviar para auditoria quando necessário.
8. Consolidar resposta final.

## Quando parar

Parar e pedir aprovação quando a tarefa envolver:

- produção
- deploy
- commit
- push
- envio externo
- campanha paga
- exclusão de arquivos
- dados financeiros
- dados pessoais
- segredo ou chave de API
